1.Команда git merge поєднує кілька послідовностей коммітів у загальну історію.

2.Найчастіше команду git merge використовують для об'єднання двох гілок.

3.Цей варіант злиття у наступних прикладах.
 У таких випадках команда git merge приймає два покажчики на комміти (зазвичай останні в гілці)
 і знаходить спільний для них батьківський коміт.

4.Потім Git створює коміт злиття, в якому поєднуються зміни з обох послідовностей, вибраних до злиття.


команда git merge часто використовується в поєднанні з командами git checkout (для вибору поточної гілки)
 та git branch -d (для видалення застарілої цільової гілки).
